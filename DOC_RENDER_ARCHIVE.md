# 功能文檔：渲染引擎與存檔機制

## 1. Jinja2 動態渲染引擎
系統目前全面採用 Jinja2 模板引擎取代傳統的字串替換邏輯。

### 優勢
*   **動態循環**：使用 `{% raw %}{% for topic in section.topics %}{% endraw %}` 語法，支援任意數量的議題產出。
*   **佈局解耦**：HTML 視覺設計 (`template.html`) 與 Python 邏輯 (`step3_render.py`) 完全分離。
*   **自適應性**：渲染器會根據 Gemini 生成的文章數量自動調整網格佈局。

## 2. 視覺化設計 (UI/UX)
*   **風格指南**：採用 **Glassmorphism (玻璃擬態)** 風格。
*   **交互效果**：支援情資卡片翻轉、流暢的平滑滾動導覽。
*   **RWD 支援**：針對不同螢幕尺寸（含行動端）進行了初步優化。

## 3. 存檔引擎
每次情報更新循環的最後一步會執行資料持久化：
1. **DB 檔案**：`intelligence.db` (供未來趨勢分析使用)。
2. **結構化紀錄**：所有欄位（含原文標題、來源連結與圖片 URL）均完整儲存於 SQLite。
